## Script Loading Strategy

### Implemented Deferred Script Loading

Moved status dashboard scripts into the document `<head>` using `defer`.

Example:

```html
<script src="/scripts/main.js" defer></script>
<script src="/scripts/status.js" defer></script>
```

Benefits:

- non-blocking page rendering
- preserved script execution order
- safe DOM access after parsing
- cleaner modern frontend structure
- improved scalability for larger applications

## Dashboard State Management

### Added Neutral Loading State System

Refactored dashboard badges to initialize in a neutral loading state instead of using hardcoded service states.

Previous behavior:

```html
class="status-badge online"
```

New behavior:

```html
class="status-badge loading"
```

Added:

```html
Checking...
```

Purpose:

- eliminate visual color flashing during refresh
- prevent incorrect initial states before API responses load
- establish API data as the single source of truth
- improve dashboard rendering consistency
- create a more production-style monitoring workflow

Dashboard flow now follows:

```text
Page Load
    ↓
Neutral Loading State
    ↓
Fetch JSON / API
    ↓
Render Final Service States
```
