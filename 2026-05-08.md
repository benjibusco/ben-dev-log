## Linux Permissions & Groups

### Shared Group Architecture

- Created shared development group: `webdev`
- Added `benji` to `webdev` using `usermod -aG webdev benji`
- Learned:
  - purpose of supplementary groups
  - meaning of `-aG`
  - difference between primary and supplementary groups
  - why Linux creates a user group matching the username

### Ownership & Permission Standardization

- Standardized active web environments using:
  - owner → `benjibusco`
  - group → `webdev`
- Applied recursive ownership updates using:
  - `chown -R benjibusco:webdev`
- Learned:
  - meaning of `-R` (recursive)
  - difference between owner/group permissions
  - how setgid inheritance works
  - why shared groups are cleaner than using `777`

### Permission Learning

- Deep dive into Linux permission numbers:
  - `755`
  - `775`
  - `664`
  - `700`
  - `600`
- Learned:
  - read/write/execute values (`4/2/1`)
  - why `777` is dangerous
  - why `775` is appropriate for collaborative environments
  - why HTML files should not be executable
- Fixed executable permissions on `index.html`
- Learned that:
  - group members can edit files
  - but only owner/root can change file permissions
